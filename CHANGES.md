# Project G Prototype — 기획서 대비 변경 이력

`index.html` (플레이 가능한 프로토타입)을 구현하면서 원본 기획서(`Doc/`)와
달라진 부분을 전부 정리한 문서입니다. 각 항목은 아래 세 가지 중 하나로 분류됩니다.

- **[판단]** 기획서가 모호하거나 상호 모순돼서 제가 근거를 대고 확정한 부분
- **[튜닝]** 기획서 수치를 그대로 구현하면 플레이가 불가능해서 의도적으로 바꾼 부분
- **[추가]** 기획서에 아예 없는, 프로토타입 편의를 위해 새로 만든 시스템
- **[버그]** 기획 의도와는 원래 같았어야 하는데 구현 과정에서 제가 실수했던 것 (참고용)

버전은 이 프로토타입이 실제로 거쳐온 수정 라운드를 순서대로 기록한 것입니다.

## v0.1 — 최초 구현

**[판단] WS(세계관 점수) 변화 공식: 가산식 채택**
인게임 사양.html §9.4-G는 곱연산(`현재값×(100+value)/100`)이라고 서술하지만,
사건.html의 컬럼 스키마 자체는 "`EffectWSGroup01[]` — **현재 해당 점수 + 변화량**"이라고
명시하고, 46개 이벤트 예시도 전부 이 서술과 일치합니다. 필드 정의 문서(사건.html)가
후반 요약 문서(인게임 사양.html)보다 원천적이라고 판단해 가산식으로 확정.

**[판단] 초기 신 가격 배율 M의 인구/신앙 ID**
§9.5-A는 "ID 4/5번 값"이라고 쓰여 있지만, 점수.html의 실제 ID는 인구수=7, 신앙=6.
문화(4)/인구밀집(5)으로 인구·신앙 배율을 계산하는 건 의미상 성립하지 않아 7/6으로 정정.

**[판단] 신.html 스킬 ID 참조 오류 정정**
데메테르/아테나/아폴론이 스킬2·3 슬롯에 자기 스킬1 ID를 그대로 반복 참조하고 있고,
헤르메스 스킬3은 `1090003`(존재하지 않는 ID)으로 적혀 있음. 스킬.html에 정의된
개별 ID(`1004002`/`1004003` 등, `1009003`)로 정정.

**[판단] 하데스 기본 스킬 영혼 임계값**
설명 텍스트는 "30개"라 하지만 데이터 필드 `SkillBaseValue01`은 `50`. 코드는
텍스트가 아닌 데이터를 읽어야 한다는 원칙에 따라 50을 채택.

**[추가] 몬스터 경로 = 타일 사이 간격을 도는 폐곡선**
인게임 사양.html §1은 "타일 간격을 몬스터가 지나다닌다"고 서술하지만 구체적 좌표계는
직접 기술하지 않음. "인클로징된 타일 블록 + 그 외곽선"을 몬스터 경로로 모델링
(5웨이브 변형 규칙 §1-C-2가 실제로 이 방식을 쓰고 있어서, 최초 생성에도 동일하게
적용 — 폐곡선·구멍 없음을 보장하는 안정적인 방법이기도 함).

**[튜닝] 몬스터 이동 속도**
웨이브/몬스터 시트의 예시 수치(BaseSpeed=2 등)를 스펙 수식 그대로 계산하면 타일
1칸을 0.02초에 통과(12칸 루프를 0.24초에 완주) — 명백히 플레이 불가능한 스텁 값.
몬스터 종류별 상대 속도(SpeedRate)는 유지하되, 한 바퀴가 웨이브 시간 내에 여러 번
돌 수 있는 합리적인 기준선(EDGE_MS_BASE=900ms/칸)으로 재조정.

## v0.2 — 전투/경로 버그 수정

**[버그] 공격 쿨다운을 매 프레임 새로 만드는 임시 객체에 저장**
`placedGods()`가 매 프레임 새 배열을 반환하는데 쿨다운 타이머를 그 위에 저장해서
매번 리셋됨 → 사실상 공격이 발동 안 함. 영속적인 타일 레코드로 이전.

**[버그] 공격 판정이 쿨다운 완료 "그 한 프레임"에만 사거리 검사**
몬스터가 사거리를 스치는 시간이 공격 주기보다 짧으면 거의 항상 놓침. 쿨다운
완료 후 대상이 나타날 때까지 매 프레임 계속 확인하도록 수정.

**[버그] 인구수(WS7) 초기값**
모드.html `DefaultWs07=100`인데 다른 6개 점수와 함께 임의로 50에서 시작하게
해놔서 초기 공격력이 전부 절반이었음. 100으로 정정.

**[튜닝] 경로 블록(blob) 크기 상한**
5웨이브 변형 로직에 반복 적용 시 무한 증식하는 걸 발견(맵의 73%까지 잠식).
기획서에 명시된 상한이 없어서, 최소 4칸 / 최대 그리드의 33%로 안정화.

## v0.3 — 경제 시스템 버그 수정

**[버그] 이벤트 #1 컬럼 밀림**
사건.html 원본 대조 중 발견. `EffectGodGroup01(5,10)`을 `EffectWSGroup01`에도
잘못 넣어서 존재하지 않는 WS ID(10)를 조회 → 예외 발생 → **루프 전체가 조용히
정지**(공격도, 웨이브 진행도 전부 멈춤). 나머지 19개 이벤트는 원본과 재대조하여
이상 없음 확인.

**[버그] 신 가격 재계산 시 D(WS 상관계수) 무력화**
사건 발생 시 "이전 D"를 WS 점수가 **이미 바뀐 뒤**에 스냅샷하고 있어서 이전=이후로
항상 같아짐 → 사건이 터져도 핵심 WS 변화가 가격에 반영 안 되고, 인구·신앙 배율
(대체로 우상향 편향)만 반영되고 있었음. 스냅샷 시점을 WS 변경 전으로 이동.

**[버그] 재계산 시 M(인구·신앙 배율)이 매번 중복 적용**
§9.5-A(최초 상장가)와 §9.5-B(사건 발생 시 재계산)의 M 정의가 다름 — 후자는
"이 사건 전후 변화 비율"인데 전자와 같은 "절대 기준" 함수를 재사용해서, 이미
M이 반영된 가격에 사건마다 M을 또 곱함 → 인구/신앙이 안 움직여도 웨이브마다
~1.5배씩 복리로 폭증. 사건 직전/직후 델타 비율로 재정의.

**[튜닝] 재앙(보스) HP**
원본 예시값(일반 몬스터의 30~50배, 웨이브5 기준 약 4200)은 초반 화력으로 시간
내 처치가 사실상 불가능 — 매 판 보스 웨이브에서 game over. 500%/800%로 하향.

**[튜닝] 웨이브 처치 보상**
원본 `BaseRewardAmount`가 전 웨이브 공통 10 고정, 일반 몬스터 보상률 100% →
1마리당 정확히 10 향화. 신 최저가 400 기준 웨이브 하나를 완벽 클리어해도 신
1주도 못 삼 — 전투 수익이 사실상 무의미. 60(6배)으로 상향.

**[버그] 신 발행 주식 상한(MaxQuantity) 미구현**
신.html에 신별 총 발행 주수 상한이 명시(제우스 등 50주, 헤르메스 20주 등)되어
있는데 구매 로직에 이 체크가 아예 없었음 — 돈만 있으면 한 신에 무한정 투자 가능해서
"신 하나만 사재기하면 게임이 깨지는" 밸런스 붕괴의 근본 원인이었음. 캡 적용.

## v0.4 — UX/시스템 추가 (기획서에 없는 것들)

**[추가] 신 1종당 동시 배치 1타일 제한 + 이동 시 스왑**
공격력 공식이 "배치 위치"가 아니라 "보유 주식 수 전체" 기준이라, 같은 신을
여러 타일에 두면 각 타일이 동일한 풀파워로 중복 발동하는 명백한 밸런스 버그가
됨(기획서엔 명시적 금지 문구 없음). 1신 1타일로 제한, 이미 배치된 신을 다른
자리로 옮길 때 그 자리에 다른 신이 있으면 서로 자리를 맞바꾸도록 처리.

**[추가] 웨이브 사이 "대기" 인터미션 + 수동 시작 버튼**
기획서는 웨이브가 끝나면 자동으로 다음 웨이브가 이어지는 구조(§2-A-4).
매매 판단 시간을 주기 위해 웨이브 종료 후 자동 진행을 멈추고, 플레이어가
"다음 웨이브 시작"을 눌러야 진행되도록 변경. 대기 중엔 사건이 안 터지므로
가격도 고정 — 스펙의 "사건 발생 시에만 가격 변동"을 오히려 더 명확하게
체감시켜주는 효과.

**[추가] 결과 화면(승리/패배) + 등급 판정**
기획서는 게임 종료 시 "끝난 웨이브를 기록"한다고만 서술(§3-A). 도달 웨이브,
처치 수, 최종 자산, 자산 증감률을 보여주고 S~C 등급 + 코멘트를 붙이는 화면은
전부 프로토타입 자체 추가 기능.

**[추가] 신탁 UI 단순화 — 무료 신탁 자동 표시**
"매 웨이브 첫 신탁은 무료"(§4-H)라는 규칙 자체는 스펙에 있지만, 버튼을 눌러야
확인 가능한 UI로 구현했던 것을 자동 표시로 변경(유료 "정밀 신탁" 버튼만 유지).

**[버그→정합성] 신탁 미리보기와 실제 발생 사건 불일치**
신탁이 보여주는 사건과 실제로 터지는 사건이 서로 다른 독립 추첨이었음. §4-D
("이번 판의 사건 시퀀스는 게임 시작 전 미리 확정")에 맞춰, 웨이브가 "대기" 상태가
되는 시점에 그 웨이브의 사건 시퀀스를 미리 한 번만 추첨해 큐에 저장하고, 신탁도
실제 발동 로직도 같은 큐를 참조하도록 통일.

**[버그] 경로에 둘러싸인 타일 구매 금지 (제 임의 규칙, 근거 없음)**
경로 구현 과정에서 "블록 안쪽 = 소유 불가"로 편의상 확장했으나 기획서 어디에도
그런 제약 없음. 모든 타일이 구매 가능하도록 되돌림(가격만 근접도로 비쌈).

**[추가] 공격력/평단가/처치 수/다음 사건 타이머 등 HUD 정보**
기획서에 언급된 수치(현재 공격력 공식, 평단가 개념 등)를 화면에 실시간으로
노출하는 UI는 전부 프로토타입 자체 추가.
